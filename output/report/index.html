🔍 How the Pipeline Works

📖 Read→ 📐 Resolve→ 🎯 Strategy→ ⚙️ Process
1. metadata-readerReads image dimensions + format via sharp, stores in cabinet
2. dimension-resolverScales to meet 1MP min area, snaps both axes to 64px ceiling, detects if ratio changes
3. strategy-agentIf ratio unchanged → fill (direct resize). If changed → contain (pad to preserve ratio)
4. image-processorExecutes sharp with chosen strategy, saves output with dimension suffix

All four agents communicate through a shared cabinet (namespaced key-value store). Each agent reads its inputs from cabinet keys written by previous agents and writes its own outputs to new cabinet keys. The Controller orchestrates the pipeline via the Sequence axiom — running agents in order with snapshot/rollback support. The Condition axiom is available for reflex checks (e.g., skip strategy-agent when ratio is preserved). The Signal axiom is available for lesson routing and cross-cutting alerts.
